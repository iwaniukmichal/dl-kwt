# Usage

Use this skill when the codebase already exists, the user already ran experiments, and the next step is to turn existing files in `outputs/` into the final report.

## Typical invocation

```text
Use the experiment-report-runner skill.
The repository is already implemented and the experiments were already run by the user.
Read plan.tex to understand the project structure, phases, and research questions.
Read README.md to identify the experiment workflow.
Inspect outputs/ and use only the files found there as evidence.
Derive cross-seed averages, cross-architecture comparisons, phase-over-phase deltas, augmentation rankings, and answers to the plan's research questions.
Write the final report into docs/report/report.tex.
Use the report template sections from docs/report/report.tex and follow latex-writing-skill for LaTeX formatting.
```

## Expected supporting files

The skill works best when the repository contains or provides:

- `docs/plan/plan.tex` (or `references/plan.tex`) — the structural guide for report content
- `docs/report/report.tex` — the report section layout/template
- `README.md` — the experiment workflow and CLI commands
- `outputs/` with experiment artifacts already generated by the user

## Key improvements over basic artifact inspection

This skill goes beyond raw data collection:

- **Derived statistics**: computes mean ± std across seeds, deltas between phases, comparisons between architectures
- **Augmentation rankings**: ranks strategies by effectiveness per architecture and per training regime
- **Research question answers**: drafts explicit answers to the plan's research questions using discovered metrics
- **Insightful tables**: every table includes an interpretation paragraph — no raw data dumps
- **Plan-driven structure**: report content follows the plan's phase structure, not an ad-hoc layout

## Notes

- This skill is for artifact inspection, computation, and report writing — not experiment execution.
- It treats `plan.tex` as the authority on what to report and how to structure the content.
- It treats `outputs/` as the sole evidence source for all quantitative claims.
- Every numeric result in the report must be a derived statistic, not a raw copied value.
- It should only suggest rerunning experiments as a separate follow-up, not the default behavior.
